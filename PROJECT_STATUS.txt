CONCEPTS LEARNED 2026-01-18
===============================================

HTTP/ARCHITECTURE CONCEPTS:

1. GET vs POST
   - GET = "give me something" (retrieve/view data)
   - POST = "take this and process it" (send data)
   - GET: data in URL, bookmarkable, cached
   - POST: data in body, not bookmarkable, not cached
   - Form method="POST" tells browser how to send data
   - Flask: request.args for GET, request.form for POST

2. Server-side vs Client-side rendering
   - Server-side: server does work, sends complete pages
   - Client-side: server sends data, browser does work
   - This app evolved from server-side to client-side calculation
   - Python now just provides data, JavaScript handles interaction

JAVASCRIPT CONCEPTS:

1. for...of loops
   - Syntax: for (let item of array) { }
   - Runs once for each item in the array
   - The variable (item) becomes each value in turn
   - Use for: iterating over arrays

2. Dynamic string building
   - "btn" + num creates "btn1", "btn2", etc.
   - Strings can be built from variables
   - document.getElementById("btn" + num) works because the result is just a string

3. Closures
   - Functions "remember" variables from when they were created
   - Each function in a loop captures its own value
   - Even after the loop ends, each function remembers its value
   - This is why 10 loop iterations create 10 different functions

4. .slice(start, end) for strings
   - Extracts a portion of a string
   - Indexes start at 0
   - Negative indexes count from end: -1 = last character
   - slice(0, -1) = everything except the last character
   - "12345".slice(0, -1) → "1234"

5. const vs let
   - const = value won't be reassigned (use for most things)
   - let = value will be reassigned (loop variables, changing values)
   - const rates = ... signals "this won't change"

6. <span> element
   - Inline element (doesn't break the line)
   - Use to mark text you want to target with JS/CSS
   - Block elements (<p>, <div>) take full width
   - Inline elements (<span>, <a>) flow with text

HTML CONCEPTS:

1. Option value attribute
   - <option value="USD">United States Dollar</option>
   - Display text = what user sees ("United States Dollar")
   - Value attribute = what code receives ("USD")
   - .value returns the value attribute, not display text
   - Without value attribute, .value returns display text

2. HTML special characters
   - ⌫ = backspace symbol
   - ← = left arrow (&larr;)
   - Can type directly or use HTML codes

DATA FLOW CONCEPTS:

1. Lookup tables (dictionaries for mapping)
   - Store related data together: {"USD": {"name": "...", "symbol": "$"}}
   - Access with: currency_info["USD"]["symbol"]
   - Use when you need to look up info by a key

2. Single source of truth
   - Keep data in one place (Python), pass to where needed
   - Better than maintaining duplicate data in multiple places
   - currency_info lives in Python, passed to Jinja and JavaScript
